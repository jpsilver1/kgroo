웹사이트를 물리적으로 그대로 긁어오는 방식(HTML 다운로더 등)은 코드가 난독화되어 있어 커스터마이징이 불가능하고 백엔드 데이터 연동도 안 됩니다. 따라서 Claude의 제안대로 "구조 분석 ➡️ AI 프롬프팅 ➡️ 컴포넌트 단위 구현"으로 가는 것이 정석입니다.
이 방향성에 맞춰 1) 가장 효과적인 복제(구현) 프로세스와 2) 작업에 사용할 AI 툴 조합 및 역할을 딱 정리해 드리겠습니다.

1. 가장 효과적인 사이트 복제 및 커스터마이징 프로세스
버틀러(Butler) 같은 고도화된 대시보드 사이트는 한 번에 다 만들려고 하면 지칩니다. 섹션별, 기능별로 쪼개서 AI에게 지시해야 합니다.
① 뼈대 및 비주얼 자산 수집
	•	전체 화면 캡처: 크롬 확장 프로그램 GoFullPage를 이용해 버틀러의 메인 홈, 스크리너, 포트폴리오 등 주요 화면을 전체 캡처합니다. (이 캡처본들이 AI에게 줄 가장 강력한 프롬프트 재료가 됩니다.)
	•	개발자 도구(F12)로 컬러/폰트 확인: 버틀러의 깔끔한 느낌을 주는 메인 배경색, 패널 색상, 폰트 크기만 슬쩍 메모해 둡니다.
② 기능 단위 분해 (MVP 수립)
	•	한 번에 전체 사이트를 만들지 말고, 아래 순서대로 화면을 하나씩 완성해 나갑니다.
	•	1단계: 랜딩 페이지 (홈 화면) 및 기본 대시보드 레이아웃 (좌측 메뉴 + 상단 검색창)
	•	2단계: 스크리너 화면 (테이블 형식의 기업 리스트)
	•	3단계: 기업 상세 페이지 (차트 시각화)
③ AI를 통한 컴포넌트 생성 및 조립
	•	캡처한 이미지와 함께 AI에게 "이 UI를 React와 Tailwind CSS로 짜줘"라고 요청하여 컴포넌트(Header, Sidebar, Card, Table 등)를 하나씩 생성하고 조립합니다.
④ 내 스타일로 커스터마이징
	•	AI가 만들어준 깔끔한 구조에 JP님만의 브랜드 컬러(예: 다크 모드 테마, 혹은 프리미엄 골드/네이비 톤 등)와 로고, 본인만의 텍스트 문구를 입힙니다.

2. 작업 단계별 추천 AI 툴 조합 및 역할
현재 개발 생태계에서 가장 성능이 뛰어난 AI 조합인 Claude 3.5 Sonnet / Claude Code와 Cursor(또는 VS Code + 전용 확장), 그리고 백엔드 특화 AI를 조합하는 것이 가장 효율적입니다.
개발 단계
추천 AI 툴
AI의 구체적인 역할 및 활용법
1. UI/UX 및 프론트엔드 개발
Claude 3.5 Sonnet

(또는 Cursor IDE)
* 역할: 화면 캡처본을 인식하여 코드로 변환.

* 방법: GoFullPage로 캡처한 이미지들을 Claude에 첨부한 뒤, *"이 대시보드 화면이랑 똑같은 레이아웃을 React + Tailwind CSS 컴포넌트로 분리해서 작성해줘"*라고 요청합니다. 테일윈드를 쓰면 스타일 수정이 극도로 쉬워집니다.
2. 복잡한 인터랙션 및 코드 수정
Claude Code

(또는 Cursor Composer)
* 역할: 로컬 파일들을 직접 수정하고 에러 해결.

* 방법: 프론트엔드 파일 구조가 잡힌 후, *"Sidebar랑 Main 패널이 반응형으로 접히게 JavaScript 물려줘"*라고 터미널이나 에디터 내에서 명령하면 전체 프로젝트 컨텍스트를 읽고 알아서 코드를 수정해 줍니다.
3. 차트 및 데이터 시각화
Claude + Recharts 라이브러리
* 역할: 버틀러의 핵심인 '재무제표 시각화 그래프' 구현.

* 방법: Claude에게 *"금융 데이터 대시보드에 쓸 차트가 필요해. Recharts 라이브러리를 사용해서 버틀러 스타일의 막대/선 혼합 차트 컴포넌트 만들어줘"*라고 요청합니다.
4. 백엔드 API 및 데이터 연동
Claude 또는 Codex

(Python FastAPI 추천)
* 역할: 주식/기업 데이터를 가져와서 프론트엔드에 쏴주는 API 서버 구축.
